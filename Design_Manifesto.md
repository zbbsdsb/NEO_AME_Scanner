AMAR Engine: The Design Manifesto
I. The Core Philosophy: Absolute Clarity
Design is the interface between chaos and logic. For the AMAR Engine (AME), UI is not a layer of decoration but a framework of order. By stripping away the non-essential, we allow the complexity of 3DGS and AI orchestration to become intuitive. We aim for a "Silent Utility"—a tool that feels as reliable as a physical instrument.

II. The Monochromatic Palette: Depth in Grayscale
We embrace a strict spectrum of Black, White, and Light Gray to establish an environment of timeless professionalism.

Optic White: Reserved for primary content and high-level typography to ensure maximum legibility.

Subtle Grays: Used for layout borders and background layering to create depth without visual noise.

True Black: Used for structural elements and navigation anchors, providing a "gravity" that grounds the digital experience.

The Absence of Color: By removing hue, we force the user to focus on structure, data, and the spatial relationships within the engine.

III. Aesthetic Pillars: Swiss Precision
The Grid: Every element adheres to a rigorous mathematical grid. Alignment is not a suggestion; it is a law.

Typography as Architecture: We use high-contrast, sans-serif typefaces. Weight and scale are the only tools used to define hierarchy, mimicking the structured elegance of Cloudflare’s technical documentation.

Micro-Borders: Instead of shadows, we define sections using 1px solid lines in light gray. This creates a "blueprint" aesthetic—precise, engineered, and clean.

IV. Interaction: The Tactile Digital
Drawing inspiration from the Dify interface, the interaction model prioritizes functional density and immediate feedback.

Instantaneous Transitions: No slow fades or "plastic" animations. Views switch with the snap of a mechanical toggle.

Hover Logic: Interactive elements react with subtle grayscale shifts. A light gray button might darken slightly upon contact, providing a silent confirmation of intent.

Modular Encapsulation: Every tool, from the AME Scanner to the LLM Configurator, is housed in a distinct, bordered container. This "containerization" reflects the modular nature of the backend.

V. The Ethos: Engineering Order
AMAR Engine is built at the intersection of Chaos Theory and Machine Intelligence. The UI must be the antidote to that complexity. We do not design for the eyes; we design for the mind.

"Eliminate the unnecessary so that the essential may speak."
Structure is Beauty. Logic is Elegance.
